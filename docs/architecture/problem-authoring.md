# Problem authoring backend

The application layer exposes authorized draft creation, MAS validation, bounded preview, generation status, cancellation, and atomic publication. Preview never publishes tests and returns the seed, runtime version, diagnostics, bounded inputs, and truncation state.

PostgreSQL mode implements `IProblemStudioStore`. The `AddProblemAuthoring` migration creates draft, localized statement metadata, generation operation, generated test set, generated test, and package import/export audit tables. Drafts and operations have optimistic concurrency tokens. A partial unique index permits one pending/running generation per draft, one test set is allowed per operation, and test indexes are unique within a set.

The authenticated `/api/problem-studio` surface supports draft creation/retrieval, author and examination assignments, localized statements, bounded assets, MAS update/validation/preview, reference-solution revisions, generation start/status/cancellation, published-test inspection, package validation, CreateNew import, and deterministic export. Owner and Editor assignments may mutate; Reviewer and Viewer assignments are read-only. ExamManager access is scoped through examination assignments. Candidates, room invigilators, evidence reviewers, and judge-worker credentials do not receive authoring access. Open examinations block mutation and publication.

Large authoring content uses generated object identifiers. Files are written atomically beneath a canonical storage root, bounded and SHA-256 hashed, and remain staged until relational publication commits. Reconciliation asks PostgreSQL which staged identifiers are referenced; referenced staged files are promoted and only stale unreferenced files are removed. A database lookup failure aborts cleanup.

Reference-output work is a separate durable PostgreSQL queue. Authenticated workers claim it with `FOR UPDATE SKIP LOCKED`, renew an unpredictable lease, download hash-verified sources and staged inputs, compile once, run every test through the fail-closed sandbox, and upload bounded hash-addressed outputs. Publication verifies complete input/output metadata and commits one published set, judge profile, operation state, and outbox message atomically; object promotion follows the relational commit and is recoverable by reconciliation.

CreateNew package import is durable and idempotent by request key. ReplaceDraft import is not implemented. Export requires the explicit hidden-test permission and uses deterministic entry order, fixed ZIP timestamps, stable JSON, and checksums. Export auditing and storage of exported package objects remain incomplete.
