# Privacy model

The deployment operator is the data controller and selects its database and object storage. The Mastemis project receives no candidate identity, source, screenshot, evidence, private contest data, authentication data, or server log through this codebase. There is no mandatory telemetry.

Before SFE starts, the operator must present a clear notice naming the operator and the monitored activity signals, event-triggered screenshot circumstances, stored source revisions, realtime invigilator alerts, evidence use, and the rule that the third confirmed violation terminates a session. Acknowledgement must be an affirmative action and must not be preselected.

Screenshots and other large evidence objects belong behind object storage. PostgreSQL should contain ownership, timestamps, identifiers, hashes, chain data, and access audits. Evidence is described as append-only at the application layer and tamper-evident, never physically impossible to modify.
